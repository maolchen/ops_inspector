# 基础资源详情表格动态列设计方案

## 一、问题分析

### 1.1 当前问题
1. **表格列硬编码**：`detail.vue` 中表格列是固定的，无法动态适应规则增减
2. **`show_in_table` 意义缺失**：规则中的 `show_in_table` 字段无法真正控制列的显示/隐藏
3. **labels 列无法展示**：如"挂载点"这类从 Prometheus labels 中提取的列，无法通过规则配置动态添加

### 1.2 期望效果
1. 增加/删除规则时，表格列自动增减
2. 可以通过规则配置，从 labels 中提取额外的列
3. 列的显示顺序可配置

## 二、数据模型设计

### 2.1 规则模型扩展

在 `Rule` 模型中新增以下字段：

```go
// backend/internal/model/rule.go

type Rule struct {
    // ... 现有字段 ...
    
    // 表格列配置
    TableColumnOrder   int    `gorm:"default:0" json:"table_column_order"`    // 列显示顺序，数字越小越靠前
    TableColumnWidth   int    `gorm:"default:100" json:"table_column_width"`  // 列宽度（像素），0表示自动
    TableColumnType    string `gorm:"size:20;default:'value'" json:"table_column_type"` // 列类型：value=规则值, label=从labels提取
    TableColumnLabel   string `gorm:"size:50" json:"table_column_label"`      // 当 table_column_type=label 时，指定要提取的label键名
    TableColumnMerge   bool   `gorm:"default:true" json:"table_column_merge"` // 是否参与单元格合并（同一IP多行时合并）
    
    // ... 其他字段 ...
}
```

### 2.2 字段说明

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `table_column_order` | int | 0 | 列显示顺序，数字越小越靠前 |
| `table_column_width` | int | 100 | 列宽度（像素），0 表示自适应 |
| `table_column_type` | string | "value" | 列类型：`value`=显示规则值，`label`=从 labels 提取 |
| `table_column_label` | string | "" | 当 `table_column_type=label` 时，指定要提取的 label 键名 |
| `table_column_merge` | bool | true | 是否参与单元格合并（同一 IP 多行时合并） |

## 三、前端实现方案

### 3.1 动态列配置生成

```typescript
// frontend/src/views/reports/detail.vue

// 获取动态列配置
const tableColumns = computed(() => {
  // 只取基础资源分组且 show_in_table 的数据
  const basicItems = items.value.filter(i => 
    isBasicResourceGroup(i.group_name) && i.show_in_table
  )
  
  // 收集所有唯一的列配置
  const columnMap = new Map<string, TableColumn>()
  
  basicItems.forEach(item => {
    const key = item.table_column_type === 'label' 
      ? `label_${item.table_column_label}` 
      : `value_${item.rule_name}`
    
    if (!columnMap.has(key)) {
      columnMap.set(key, {
        prop: key,
        label: item.table_column_type === 'label' 
          ? item.table_column_label 
          : item.rule_name,
        width: item.table_column_width || 100,
        order: item.table_column_order || 0,
        type: item.table_column_type,
        merge: item.table_column_merge,
        unit: item.unit
      })
    }
  })
  
  // 按顺序排序
  return Array.from(columnMap.values()).sort((a, b) => a.order - b.order)
})
```

### 3.3 数据行生成逻辑

```typescript
// 创建基础资源行数据（动态版本）
const createBasicRow = (
  ip: string, 
  data: Record<string, any>, 
  mountpoint: string, 
  isFirst: boolean,
  columns: TableColumn[]
) => {
  const row: Record<string, any> = {
    _ip: ip,
    _isFirst: isFirst,
    ip: isFirst ? ip : ''
  }
  
  // 根据列配置动态填充数据
  columns.forEach(col => {
    if (col.type === 'label') {
      // 从 labels 提取
      const firstItem = Object.values(data)[0]
      const labels = firstItem?.labels || {}
      row[col.prop] = labels[col.label] || '-'
    } else {
      // 规则值
      const item = data[col.label] // col.label 实际是规则名
      if (item) {
        row[col.prop] = item.value
        row[`${col.prop}_status`] = item.status
      } else {
        row[col.prop] = '-'
      }
    }
    
    // 非首行的合并列数据清空
    if (!isFirst && col.merge) {
      row[col.prop] = ''
    }
  })
  
  return row
}
```

### 5.1 迁移脚本

```sql
-- 为 rules 表新增字段

ALTER TABLE rules ADD COLUMN table_column_type VARCHAR(20) DEFAULT 'value';
ALTER TABLE rules ADD COLUMN table_column_label VARCHAR(50) DEFAULT '';
ALTER TABLE rules ADD COLUMN table_column_merge BOOLEAN DEFAULT true;

-- 为 inspection_items 表新增字段
ALTER TABLE inspection_items ADD COLUMN table_column_order INTEGER DEFAULT 0;
ALTER TABLE rules ADD COLUMN table_column_width INTEGER DEFAULT 100;
ALTER TABLE inspection_items ADD COLUMN table_column_type VARCHAR(20) DEFAULT 'value';
ALTER TABLE inspection_items ADD COLUMN table_column_label VARCHAR(50) DEFAULT '';
ALTER TABLE inspection_items ADD COLUMN table_column_merge BOOLEAN DEFAULT true;

-- 初始化现有规则的列配置
UPDATE rules SET 
  table_column_order = CASE name
    WHEN 'CPU核心数' THEN 10
    WHEN 'CPU使用率' THEN 20
    WHEN '运行时间' THEN 30
    WHEN '5分钟负载' THEN 40
    WHEN '内存总量' THEN 50
    WHEN '内存使用量' THEN 60
    WHEN '内存使用率' THEN 70
    WHEN '磁盘总量' THEN 90
    WHEN '磁盘使用量' THEN 100
    WHEN '磁盘使用率' THEN 110
    WHEN 'TCP连接数' THEN 120
    WHEN 'TCP_TW数' THEN 130
    ELSE 200
  END,
  table_column_width = CASE name
    WHEN '运行时间' THEN 130
    WHEN 'CPU使用率' THEN 110
    WHEN '内存使用率' THEN 110
    WHEN '磁盘使用率' THEN 110
    ELSE 100
  END
WHERE show_in_table = true;
```

## 九、注意事项

1. **向后兼容**：新增字段都有默认值，不影响现有数据
2. **性能考虑**：动态列在 computed 中生成，有缓存机制
3. **单元格合并**：`table_column_merge` 字段控制合并行为，磁盘相关列设为 false
4. **labels 提取**：当 `table_column_type=label` 时，query 字段可为空（仅用于配置展示）

### 使用说明

1. **编辑现有规则**：进入规则管理页面，编辑需要展示在表格中的规则
2. **开启表格展示**：打开"表格展示"开关
3. **配置列属性**：
   - 设置"列顺序"（数字越小越靠前）
   - 设置"列宽度"（像素，0 表示自适应）
   - 选择"列数据来源"：
     - `规则查询值`：显示该规则查询的结果值
     - `从标签提取`：显示 Prometheus labels 中的指定标签值
   - 设置"合并单元格"：
     - 开启：同一 IP 多行时合并显示（适用于非磁盘指标）
     - 关闭：每行独立显示（适用于挂载点等磁盘相关列）
4. **执行巡检**：新建巡检报告，新的列配置会在报告中生效

### 挂载点列配置示例

如需添加"挂载点"列：

1. 新建规则（或修改现有磁盘相关规则）
2. 配置如下：
   - 规则名称：`挂载点`
   - 表格展示：开启
   - 列数据来源：`从标签提取`
   - 标签键名：`mountpoint`
   - 列顺序：`10`
   - 合并单元格：关闭
